> [!note] 
> VASP 的 `OUTCAR` 文件包含了所有受力信息，但它太长，直接打开看比较麻烦。可以直接在服务器终端使用以下几个命令，实现边算边看。 

### 一、 电子步（SCF）监控：防范“假死”与震荡

结构优化的每一次“移动”（离子步），都建立在电子云正确分布（电子步收敛）的基础上。如果电子步算崩了，受力必然是全错的。
#### 1. 基础心跳监测：看能量变化

在任务运行的目录下，输入以下命令：

Bash
```
tail -f OSZICAR
```

- **作用**：这会让屏幕实时滚动显示 VASP 的计算步数和能量变化（`dE`）。一旦看到 `dE` 变成了正数且越来越大，或者 `dE` 出现 `******`，就说明系统快要爆炸了，可以立刻停止并 `kill` 掉任务，及时止损。
- 另外，在 `OSZICAR` 的输出中，每一行 `DAV:` 后面都有 `rms` 和 `rms(c)` 两个值。它们代表上一次迭代与这一次迭代之间，电荷密度的“差异度”。
-  正常的收敛过程，<mark style="background:rgba(160, 204, 246, 0.55)"><mark style="background:rgba(160, 204, 246, 0.55)">`rms` 应该像下楼</mark>梯一样，数量级稳步变小</mark>（例如从 `0.1` 降到 `0.001` 再降到 `0.1E-04`）。如果你盯着屏幕，发现 `rms` 的值在 `0.1E-02` 和 `0.5E-02` 之间反复横跳（像心律失常一样），这就是典型的**电荷震荡（Charge Sloshing）**。这在掺杂了大尺寸 P 原子的磁性体系中极易发生，看到这种现象，说明计算已经陷入死循环了。

    `当我们在终端输入 tail -f OSZICAR 后，屏幕会被“锁住”并持续滚动刷新最新计算的数据。要退出这个动态监控模式，直接在键盘上同时按下 Ctrl + C。
    底层原理：在 Linux 系统中，Ctrl + C 的作用是向当前正在前台运行的程序发送一个中断信号 (SIGINT)。这会立刻终止 tail -f 这个显示命令的运行，并回到熟悉的输入提示符（例如 [user@node ~]$）下。`

#### 2. 监控电子步是否“触顶” (Hitting NELM)

**命令：**
Bash
```
grep "DAV:" OSZICAR | tail -n 15
```   

-  `INCAR` 中设置了 `NELM = 200`（电子步最大迭代次数）。正常情况下，几十步内电子步就该收敛并跳到下一个离子步。
    
- **怎么看：** 观察命令输出的 `DAV:` 后面的数字。如果这个数字一直飙升，直接顶到了设置的 `200`，说明电子云处于“疯狂震荡”状态，根本找不到平衡点。这通常意味着初始结构极度不合理，或者 `AMIX/BMIX` 阻尼参数没压住。**此时不要犹豫，立刻终止任务。**

### 二、 磁矩保卫战

结合我的研究（Fe-Co 双原子的高自旋/强轨道耦合），磁矩（Magnetic Moment）是我的核心参数。如果在弛豫过程中磁矩突然消失了，即使最后结构收敛了，这大概率也是一个毫无物理意义的“假基态”。

- **命令：**

Bash

```
grep "mag=" OSZICAR | tail -n 10
```

- 我的 `INCAR` 里给 Co 设置了 `3.0` 的初始磁矩（钴磁态猜测）。VASP 在每算完一个离子步后，会在 `OSZICAR` 里打印当前体系的总磁矩（`mag=` 后的数值）。
    
- **怎么看：** 在优化的前几步，磁矩发生变化是完全正常的（因为原子在移动，晶体场在变）。但如果总磁矩突然**断崖式掉到 `0.000` 附近并持续淬灭**，这就说明 VASP 没能保住高自旋态，电子全配对了。这时候需要立刻停掉，回去检查 `INCAR` 里的 `MAGMOM` 是否设得不够强，或者是否需要配合 `LDAU` 进一步干预。

### 三、 隐形刺客监控：捕获 VASP 的底层警告

VASP 是个很“隐忍”的程序，有时候遇到一些物理上很危险的问题，它不会直接报错退出，而是默默地在 `OUTCAR` 里写一句警告，然后继续胡乱算下去。

- **命令：**

Bash
```
grep -i "warning" OUTCAR
```   
- **怎么看：** 这条命令会把 `OUTCAR` 里所有带有 "warning" 字眼的行抽出来。
    
    - 如果我们看到 `WARNING: DENTET: can't reach specified precision`，这说明电子自洽精度出大问题了，算出来的能量不可信。
        
    - 如果我们看到关于 `EDWAV` 或 `aliasing errors` 的警告，可能意味着截断能（`ENCUT`）与原子的赝势不匹配。

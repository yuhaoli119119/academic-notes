### 一、 创建脚本

#### 第一步：创建脚本文件（以[[计算相关/脚本的创建与使用（示例）/过渡态路径整合(pathway)\|pathway]]为例）

在终端中输入以下命令创建一个 Python 文件：

Bash
```
nano neb_traj_merger.py
```

#### 第二步：复制并粘贴以下代码

将下面的代码完整粘贴到终端中（在 `nano` 编辑器中通常使用右键或 `Shift+Insert` 粘贴）：
```
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

import os
import numpy as np

def read_poscar(filepath):
    """解析 POSCAR/CONTCAR 文件并提取晶格、元素及坐标信息"""
    with open(filepath, 'r') as f:
        lines = f.readlines()

    # 提取缩放系数和晶格矢量 (转换为真实笛卡尔矩阵)
    scale = float(lines[1].strip())
    lattice = np.array([list(map(float, line.split())) for line in lines[2:5]]) * scale

    # 提取元素种类和对应的原子数量 (VASP 5+ 标准格式)
    elements = lines[5].split()
    counts = list(map(int, lines[6].split()))

    # 判断是否存在 Selective dynamics (选择性动力学) 标识
    coord_start = 7
    has_selective = False
    if lines[7].strip()[0].lower() == 's':
        coord_start = 8
        has_selective = True

    # 读取坐标类型 (Direct or Cartesian)
    coord_type = lines[coord_start].strip()
    coord_start += 1 # 坐标数据正式开始的行号

    # 提取纯坐标数据 (只取前三列 x, y, z)
    coords = []
    for i in range(sum(counts)):
        parts = lines[coord_start + i].split()
        coords.append([float(x) for x in parts[:3]])

    return {
        'lines': lines,
        'lattice': lattice,
        'elements': elements,
        'counts': counts,
        'coord_start': coord_start,
        'has_selective': has_selective,
        'coords': np.array(coords)
    }

def main():
    print("正在解析初末态结构...")
    # 1. 读取初态和末态结构 (按照您的规则，均为 POSCAR)
    s00 = read_poscar(os.path.join('00', 'POSCAR'))
    s05 = read_poscar(os.path.join('05', 'POSCAR'))

    # 2. 定位 Li 原子的坐标索引范围
    if 'Li' not in s00['elements']:
        print("错误：在 00/POSCAR 中未找到 'Li' 元素，请检查文件格式是否为 VASP5 标准！")
        return
        
    li_index = s00['elements'].index('Li')
    li_start = sum(s00['counts'][:li_index])
    li_end = li_start + s00['counts'][li_index]

    # 3. 自动识别发生迁移的 Li 离子 (引入 PBC 校正)
    max_dist = 0.0
    target_atom_idx = -1

    for i in range(li_start, li_end):
        # 计算分数坐标差值
        d_direct = s05['coords'][i] - s00['coords'][i]
        # 核心：PBC 最小意象约定校正，处理跨边界问题
        d_direct -= np.round(d_direct) 
        # 转换为实际的笛卡尔位移向量
        d_cart = np.dot(d_direct, s00['lattice'])
        # 计算欧几里得距离 (位移量)
        dist = np.linalg.norm(d_cart)

        if dist > max_dist:
            max_dist = dist
            target_atom_idx = i

    print(f"成功锁定！发生迁移的 Li 离子序号为: 第 {target_atom_idx + 1} 号原子 (相对位移量: {max_dist:.4f} 埃)")

    # 4. 提取中间态 (01-04 CONTCAR) 和末态 (05 POSCAR) 的轨迹坐标
    print("正在提取迁移轨迹...")
    trajectory_coords = []
    # 严格按照您的文件规则
    folders_files = [('01', 'CONTCAR'), ('02', 'CONTCAR'), ('03', 'CONTCAR'), ('04', 'CONTCAR'), ('05', 'POSCAR')]

    for folder, file in folders_files:
        path = os.path.join(folder, file)
        s = read_poscar(path)
        trajectory_coords.append(s['coords'][target_atom_idx])

    # 5. 重组并写入新的 Li-POSCAR
    print("正在生成包含轨迹的 Li-POSCAR...")
    with open('Li-POSCAR', 'w') as f:
        # 写入头部信息 (标题、缩放、晶格、元素)
        for i in range(6): 
            f.write(s00['lines'][i])
            
        # 修改原子数量行 (Li 原子数 + 5)
        new_counts = s00['counts'].copy()
        new_counts[li_index] += 5
        f.write("  " + "  ".join(map(str, new_counts)) + "\n")

        # 写入动力学标识和坐标类型 (Direct/Cartesian)
        if s00['has_selective']:
            f.write(s00['lines'][s00['coord_start'] - 2])
        f.write(s00['lines'][s00['coord_start'] - 1])

        # 写入坐标体，并在原来的 Li 原子块末尾插入 5 个轨迹坐标
        for i in range(len(s00['coords'])):
            f.write(s00['lines'][s00['coord_start'] + i])
            # 当刚好写完最后一个原始 Li 原子时，追加插入轨迹点
            if i == li_end - 1:
                for idx, traj_coord in enumerate(trajectory_coords):
                    # 默认使用 Direct 坐标格式输出，保持小数点后 6 位精度
                    # 如果有 Selective dynamics 标识，默认给轨迹点加上 F F F (可选)
                    flag = "  F  F  F" if s00['has_selective'] else ""
                    f.write(f"  {traj_coord[0]:.6f}  {traj_coord[1]:.6f}  {traj_coord[2]:.6f}{flag}  # Traj_Image_{idx+1:02d}\n")

    print("\n任务完成！已生成文件：Li-POSCAR。您可以将其下载并在 VESTA 中打开查看。")

if __name__ == '__main__':
    main()
```

#### 第三步：保存并运行

1. 在 `nano` 中按 `Ctrl + O` 保存文件，按 `Enter` 确认文件名，然后按 `Ctrl + X` 退出编辑器。

2. 在终端中直接运行该脚本：

Bash

```
python3 neb_traj_merger.py
```

脚本运行后，会在屏幕上打印出识别到的迁移离子序号及其位移量，并生成一个 `Li-POSCAR` 文件。您将这个文件导入 VESTA 后，可以看到 6 个靠得很近的 Li 原子，它们连起来就是解离的过程。

### 二、 脚本名称简化

在科研高频的数据处理工作中，将冗长、难记的执行代码简化为符合直觉的自定义命令（如 `pathway`），这是非常高效且专业的做法。

在 Linux 服务器中，要实现“**在任意带有 NEB 数据的文件夹中，只需输入 `pathway` 就能自动调用该脚本**”，最稳定且一劳永逸的方法是利用 Linux 的 **“别名 ”** 功能配合**绝对路径**。
配置自定义命令 `pathway` 的分步指南

在 Linux 终端中依次执行以下步骤：
#### 第一步：建立个人的集中式脚本库

为了防止脚本在各个计算目录中混淆或误删，我们先在您的用户根目录（主目录）下创建一个专门存放脚本的文件夹。

Bash

```
mkdir -p ~/my_scripts
```

_注：`~` 符号在 Linux 中代表您的个人主目录（如 `/home/username/`）。_

#### 第二步：将脚本移动到该集中库

请确保您目前所在的路径是刚才生成 `neb_traj_merger.py` 脚本的目录，然后执行移动命令：

Bash

```
mv ~/neb_traj_merger.py ~/my_scripts/
```

#### 第三步：绑定 `pathway` 别名到系统环境变量中

我们需要告诉 Linux 操作系统：“以后当我输入 `pathway` 时，请等同于执行 `python3 ~/my_scripts/neb_traj_merger.py` 这个绝对路径下的命令。”

我们将这句指令写入到 `~/.bashrc` 文件中。(`~/.bashrc` 是 Linux 用户的环境配置文件，每次您登录服务器时，系统都会悄悄读取它)。 请在终端直接复制并粘贴这行代码后回车：

Bash

```
echo "alias pathway='python3 ~/my_scripts/neb_traj_merger.py'" >> ~/.bashrc
```

_注：`>>` 表示将内容追加到文件末尾，这样不会破坏您服务器原有的其他配置。_

#### 第四步：刷新配置使其立即生效

写入配置文件后，当前运行的终端还不知道这个变化。通过 `source` 命令重新加载配置：

Bash

```
source ~/.bashrc
```

### 三、 脚本迁移

<font color="#f79646">核心概念：脚本的可移植性与环境独立性</font>

- **跨平台文本 (Cross-platform Text)**：刚才编写的 `neb_traj_merger.py` 本质上只是一个纯文本文件。Python 属于**解释型语言**，它不像 C++ 或 Fortran 那样需要提前编译成特定服务器硬件才能识别的二进制机器码。因此，这个脚本在任何服务器上都能被读取。
    
- **局部配置局限 (Local Environment Constraint)**：脚本文件本身虽然可以随便拷贝，但是我们刚刚在 `~/.bashrc` 文件中为您设置的 **`pathway` 快捷命令（Alias 别名）**，是与当前这台服务器的“个人账户”深度绑定的。它**不会**跟着脚本一起转移。

#### 跨服务器部署脚本的 标准操作流程 (SOP)

为了在其他 Linux 服务器上完美复现“一键运行 `pathway`”的丝滑体验，请在新服务器上遵循以下分步指南：

##### 步骤一：物理转移脚本文件

可以将 `neb_traj_merger.py` 当作普通文本文件对待。

- **方法 A（最直观）**：使用常用的连接软件（如 Xshell 附带的 Xftp、WinSCP 或 MobaXterm），将文件从当前服务器下载到个人电脑桌面，然后再上传到**新服务器**的 `~/my_scripts/` 文件夹中（如果没有这个文件夹，请在新服务器上 `mkdir -p ~/my_scripts` 创建一个）。

Bash

```
mkdir -p ~/my_scripts
```

- **方法 B（最简单）**：直接在新服务器上输入 

Bash

```
nano ~/my_scripts/neb_traj_merger.py
```
  
  然后把我们之前的代码复制粘贴进去，保存即可。

##### 步骤二：重塑 `pathway` 灵魂（重写系统配置）

既然脚本已经就位，现只需要在新服务器上重复一次绑定快捷命令的操作即可。 请在新服务器终端中依次执行以下两行代码：

Bash

```
echo "alias pathway='python3 ~/my_scripts/neb_traj_merger.py'" >> ~/.bashrc
```

Bash

```
source ~/.bashrc
```

完成这二步后，您的这套自动化工具就在新服务器上成功“安家”了。以后无论您换到多少台服务器，只需照搬这个流程，就能立刻拥有您熟悉的 `pathway` 命令。

### 四、 进阶原生方案（彻底实现「文件名 = 命令名」，无需别名）

直接创建**无后缀的`pathway`文件**，做成 Linux 系统原生可执行程序，配置完成后，任何目录直接输入`pathway`就能运行，无需别名，更符合 Linux 系统的原生使用习惯，一劳永逸。

#### 分步操作

1. **直接创建无后缀的`pathway`文件**

bash

运行

```
nano pathway
```

1. **粘贴原完整代码，重点确认首行配置**
    
    原代码第一行`#!/usr/bin/env python3`（称为 shebang）必须保留且不能写错，这是系统识别「用 Python 解释器执行该文件」的核心，原代码已自带，无需修改。粘贴完成后保存退出。
2. **给文件添加可执行权限（关键步骤，缺了必报错）**

Linux 系统默认不给文本文件执行权限，必须手动添加：

bash

运行

```
chmod +x pathway
```

3. **移动到集中脚本库**

bash

运行

```
mkdir -p ~/my_scripts   
```

```
mv pathway ~/my_scripts/
```



1. **把脚本库加入系统全局环境变量 PATH（一劳永逸）**

让系统在任何目录都能找到你的`pathway`命令，执行以下命令写入配置：

bash(若该目录之前已经加入过则跳过该步骤)

```
echo 'export PATH="$HOME/my_scripts:$PATH"' >> ~/.bashrc
```
   
2. **刷新配置生效**
   
bash

```
source ~/.bashrc
```


#### 优势

- 彻底实现「文件名 = 命令名」，无需别名绑定，输入`pathway`直接运行
- 一次配置 PATH 后，后续所有放到`~/my_scripts/`里的可执行脚本，都能直接用文件名调用，扩展性极强
- 跨服务器迁移更省事，只需拷贝`pathway`文件到新服务器的`~/my_scripts/`，加一次 PATH 配置即可，不用逐个脚本配别名

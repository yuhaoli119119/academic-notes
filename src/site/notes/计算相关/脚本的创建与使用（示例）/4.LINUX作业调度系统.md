> [!note] 核心目的
> 针对实验室本地服务器“无法自动连轴转”、“多用户互相抢占资源”、且“断开终端即停止计算”的痛点，本系统利用 Linux 原生脚本实现了一个零依赖的轻量级排队调度器。
> - **全自动连轴转**：服务器 24 小时满载，自动按顺序调用 48 核执行任务。
> - **超算级体验**：通过 `vsubmit`（提交）、`bjobs`（查看）、`bkill`（终止）三个简单命令管理任务。

## 第一阶段：管理员全局部署（仅需一次）

> ⚠️ **前提条件**：以下操作必须具有最高权限。请打开终端，输入 `su` 并输入密码，全程在 **root 账号** 下执行。

### 第一步：创建全局公共队列大本营

创建一个所有用户都有权读写的中转站，用于存放排队凭证。

Bash

```
mkdir -p /var/opt/vasp_queue
chmod 777 /var/opt/vasp_queue
```

### 第二步：一键生成四大核心调度脚本

直接在终端中依次复制粘贴以下四大代码块并回车，系统会自动在 `/usr/local/bin/` 下生成无后缀的全局命令文件。

**1. 生成 `vsubmit` (提交通道)：**

Bash

```
cat << 'EOF' > /usr/local/bin/vsubmit
#!/bin/bash
QUEUE_DIR="/var/opt/vasp_queue"
if [ ! -d "$QUEUE_DIR" ]; then echo "❌ 队列目录不存在，请联系管理员部署。"; exit 1; fi
TIMESTAMP=$(date +%Y%m%d_%H%M%S_%N)
JOB_FILE="$QUEUE_DIR/${TIMESTAMP}_${USER}.job"
echo "$PWD" > "$JOB_FILE"
echo "✅ 任务已成功加入队列！"
echo "📂 计算路径: $PWD"
EOF
```

**2. 生成 `bjobs` (队列与耗时监控)：**

Bash

```
cat << 'EOF' > /usr/local/bin/bjobs
#!/bin/bash
QUEUE_DIR="/var/opt/vasp_queue"
RUN_INFO="$QUEUE_DIR/running.info"
echo "📊 VASP 服务器任务监控"
echo "------------------------------------------------"
if [ -f "$RUN_INFO" ]; then
    read r_user r_path r_start < "$RUN_INFO"
    now=$(date +%s); diff=$((now - r_start))
    echo "🏃 [正在运行] 👤 $r_user | ⌛ 已耗时: $((diff/3600))h $(((diff%3600)/60))m $((diff%60))s"
    echo "📂 路径: $r_path"
else
    echo "💤 [当前空闲] 没有任务正在计算。"
fi
echo "------------------------------------------------"
echo "⏳ [等待队列]"
count=0
for job in $(ls -tr "$QUEUE_DIR"/*.job 2>/dev/null); do
    ((count++)); user=$(echo "$job" | awk -F'_' '{print $NF}' | sed 's/\.job//'); path=$(cat "$job")
    echo "[$count] 👤 $user | 📂 $path"
done
if [ $count -eq 0 ]; then echo "✨ 目前没有其他人在排队。"; fi
echo "------------------------------------------------"
EOF
```

**3. 生成 `bkill` (精准交互终止工具)：**

Bash

```
cat << 'EOF' > /usr/local/bin/bkill
#!/bin/bash
QUEUE_DIR="/var/opt/vasp_queue"
echo "🔍 正在检查属于您 ($USER) 的任务..."
echo "----------------------------------------"

shopt -s nullglob
QUEUED_JOBS=("$QUEUE_DIR"/*_"${USER}".job)
shopt -u nullglob

if [ ${#QUEUED_JOBS[@]} -gt 0 ]; then
    echo "⏳ 您当前有 ${#QUEUED_JOBS[@]} 个任务正在排队："
    for i in "${!QUEUED_JOBS[@]}"; do
        path=$(cat "${QUEUED_JOBS[$i]}")
        echo "  [$((i+1))] 📂 $path"
    done
    echo "  [all] 💥 删掉所有排队任务"
    echo "  [0]   ⏭️ 不删除排队任务，直接跳过"
    read -p "👉 请输入要取消排队的任务编号 (例如 1, 2, all 或 0): " choice

    if [ "$choice" = "all" ]; then
        rm -f "${QUEUED_JOBS[@]}"; echo "✅ 已清除您所有排队的任务。"
    elif [[ "$choice" =~ ^[0-9]+$ ]] && [ "$choice" -gt 0 ] && [ "$choice" -le "${#QUEUED_JOBS[@]}" ]; then
        idx=$((choice-1)); rm -f "${QUEUED_JOBS[$idx]}"; echo "✅ 已精准移除任务 [$choice]。"
    elif [ "$choice" = "0" ]; then
        echo "⏭️ 已跳过排队清理。"
    else
        echo "❌ 无效输入，未删除任何排队任务。"
    fi
else
    echo "ℹ️ 您目前没有正在排队的任务。"
fi
echo "----------------------------------------"

if pgrep -u "$USER" vasp > /dev/null; then
    RUN_INFO="$QUEUE_DIR/running.info"
    if [ -f "$RUN_INFO" ]; then
        read r_user r_path r_start < "$RUN_INFO"
        if [ "$r_user" = "$USER" ]; then echo "⚠️ 发现正在运行的任务: $r_path"; fi
    fi
    read -p "👉 是否立即强行终止它？(y/n): " confirm
    if [ "$confirm" = "y" ] || [ "$confirm" = "Y" ]; then
        pkill -u "$USER" vasp; pkill -u "$USER" mpirun
        echo "🛑 运行中的任务已成功强杀！下一个排队任务将自动接替。"
    else
        echo "✅ 已取消，任务继续运行。"
    fi
else
    echo "ℹ️ 当前没有属于您的 VASP 计算在运行。"
fi
EOF
```

**4. 生成 `vdaemon` (核心调度守护进程)：**

> [!attention] 注意！代码中的 `mpirun -np 48 vasp` 是基于 54 核服务器预留 6 核的配置。如果在其它核数的服务器上配置，请将 `48` 修改为对应的合理核心数。

Bash

```
cat << 'EOF' > /usr/local/bin/vdaemon
#!/bin/bash
QUEUE_DIR="/var/opt/vasp_queue"
LOG_FILE="/var/opt/vasp_queue/daemon_sys.log"
RUN_INFO="$QUEUE_DIR/running.info"

echo "🚀 VASP 队列守护进程已启动... 日志记录于 $LOG_FILE" >> "$LOG_FILE"

while true; do
    JOB_FILE=$(ls -tr "$QUEUE_DIR"/*.job 2>/dev/null | head -n 1)
    if [ -n "$JOB_FILE" ]; then
        TARGET_DIR=$(cat "$JOB_FILE")
        JOB_USER=$(echo "$(basename "$JOB_FILE")" | awk -F'_' '{print $NF}' | sed 's/\.job//')
        rm -f "$JOB_FILE"
        if [ -d "$TARGET_DIR" ]; then
            echo "$JOB_USER $TARGET_DIR $(date +%s)" > "$RUN_INFO"; chmod 666 "$RUN_INFO"
            
            # 绝杀逻辑：切回提交者账号执行，完美规避 root 的 MPI 限制并继承环境
            su - "$JOB_USER" -c "cd '$TARGET_DIR' && ulimit -s unlimited && mpirun -np 48 vasp > vasp_run.log 2>&1"
            
            rm -f "$RUN_INFO"
        fi
    else
        sleep 30
    fi
done
EOF
```

### 第三步：赋予最高权限并启动系统

依然在 `root` 账号下，执行以下命令使所有脚本具备全局可执行权限，并挂载守护进程，使其在后台 24 小时监控：

Bash

```
chmod 755 /usr/local/bin/vsubmit
chmod 755 /usr/local/bin/bjobs
chmod 755 /usr/local/bin/bkill
chmod 755 /usr/local/bin/vdaemon

# 杀掉以前可能存在的旧进程，并挂载最新守护进程
pkill -f vdaemon
nohup /usr/local/bin/vdaemon > /dev/null 2>&1 &
```

部署完成！直接输入 `exit` 退出 root 账号即可。

# 计算服务器账户创建与环境初始化教程

 > [!note] 
>  **核心目的**：规范化服务器新用户的开户流程，并指导新用户在首次登录后，迅速搭建起如Linux 原生命令般顺畅的“个人全局脚本库”，为后续的高通量 VASP 计算打下坚实基础。

## 第一阶段：管理员为新成员创建账户 (需 Root 权限)

这一阶段需要在服务器的最高管理员（Root）账户下进行操作。Root 相当于 Windows 系统中的超级管理员，拥有对服务器任意文件和配置的最高修改权限。

### 步骤 1：登录 Root 用户

在终端中通过 SSH 登录服务器，并切换到 root 用户。

### 步骤 2：创建新用户主目录与系统身份

Linux 的多用户管理非常严谨。我们需要为新用户（例如命名为 `test`）分配一个专属的存储空间（Home 目录）。

Bash

```
useradd -d /public/home/test test
```

- **术语解析**：
    
    - `useradd`：Linux 系统中用于添加新用户的核心命令。
        
    - `-d /public/home/test`：`-d` 代表 directory（目录）。在高性能计算（HPC）集群中，用户的个人文件夹通常挂载在一个所有计算节点都能访问的共享存储路径下（如 `/public/home/`），而不是单机默认的 `/home/`。
        
    - 最后的 `test`：这是该用户在系统中的唯一登录名。
        
    - **举一反三**：如果课题组新来了一位名叫 yoso 的同学，只需将命令替换为 `useradd -d /public/home/yoso yoso` 即可。
        

### 步骤 3：为新用户设置密码

Bash

```
passwd test
```

输入该命令后，系统会提示你输入两次新密码。

- **⚠️ 注意事项**：在 Linux 终端中输入密码时，**屏幕上不会显示任何星号或字符**。这是一种安全机制，防止旁人通过字符长度猜测密码。请放心敲击键盘，<font color="#f79646">输入完毕后按回车键（Enter），然后盲打第二次以确认。</font>
    

### 步骤 4：将用户凭证同步至所有计算节点（极度关键）

Bash

```
make -C /var/yp 
```

- **术语解析**：服务器采用了 **NIS (Network Information Service)** 架构来管理集群。简单来说，集群由一个“管理节点”和数十个“计算节点”组成。刚才创建的账号只存在于管理节点上。执行 `make -C /var/yp` 是通知系统重新编译网络信息数据库，把新用户 `test` 的账号密码“广播”并同步到所有的计算节点上。**如果不执行这一步，用户提交的 VASP 任务将无法在计算节点上正常运行**。
  
  
  ## 第二阶段：新用户个人环境与脚本库配置

新用户获取账号密码并首次登录后，建议立刻进行“个人全局脚本库”的初始化。这能大幅提升后续处理 VASP 数据（如提取能量、整合 NEB 过渡态路径）的效率。

### 步骤 1：建立个人集中脚本库

为了防止脚本在各个计算目录中混淆或误删，建议在用户主目录下创建一个专属的脚本库文件夹（如 `my_scripts`）。

Bash

```
mkdir ~/my_scripts
```

### 步骤 2：配置全局环境变量 (PATH)

要想让系统在任何目录下都能像识别 `cd`、`ls` 那样直接识别并运行自定义的计算辅助脚本，我们需要将刚才创建的脚本库加入到系统的全局环境变量 `$PATH` 中。

请在终端执行以下命令（一劳永逸，以后添加新脚本无需重复此步）：

Bash

```
echo 'export PATH=$PATH:~/my_scripts' >> ~/.bashrc
```

随后，刷新配置文件使其立即生效：

Bash

```
source ~/.bashrc
```

### 步骤 3：制作并植入全局脚本

配置好环境后，后续无论是编写 Python 还是 Bash 辅助脚本，只需遵循以下三个铁律，即可实现“文件名 = 命令名”的原生调用体验：

1. **去后缀化命名**：创建脚本文件时，不要加 `.py` 或 `.sh` 后缀，直接将文件名设为想要的快捷命令（例如提取能量脚本直接命名为 `energyextract`，过渡态整合脚本命名为 `pathway`）。
    
2. **声明解释器 (Shebang)**：脚本的第一行必须指明由什么程序来运行它。
    
    - Python 脚本第一行必须是：`#!/usr/bin/env python3`。
        
    - Bash 脚本第一行必须是：`#!/bin/bash`。
        
3. **赋予执行权限 (最易错点)**：Linux 系统默认出于安全考虑，不会给新创建的纯文本文件执行权限，必须手动添加。
    
    Bash
    
    ```
    chmod +x ~/my_scripts/你的脚本名
    ```
    
 > [!attention] 
>  专家防坑指南：Windows 编码乱码问题** 如果你的脚本代码是在 Windows 系统（如 Notepad 记事本）下编辑后上传到服务器的，文件可能被保存为 `GBK` 编码格式。这在强制要求 `UTF-8` 编码的 Linux 环境中会导致乱码甚至报错。 此时，可使用以下命令转换文件格式：

 Bash> 
```
iconv -f GBK -t UTF-8 你的脚本名 -o 你的脚本名
```

 它的原理是：把该文件从 GBK 编码读取，转换为 UTF-8，然后重新输出并覆盖原文件。

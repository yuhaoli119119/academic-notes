# Linux 自定义脚本全局命令化教程（进阶原生方案）

> [!note] 核心目的
> 在科研计算中，我们常需要运行自己编写的脚本（如 `.py` 或 `.sh` 文件）。通过本教程的方法，可以将复杂难记的执行命令（如 `python3 neb_traj_merger.py`）彻底简化为一个**自定义的全局命令**（如 `pathway`），使其像 Linux 系统原生的 `cd`、`ls` 命令一样，在任何目录下都能直接调用。

### 一、 前提要求：脚本首行配置（Shebang）

要让 Linux 自动识别脚本语言，脚本文件的**第一行**必须包含解释器声明（Shebang）。
例如，如果是 Python 脚本，第一行必须是：
````
#!/usr/bin/env python3
````

_(注：如果已有该行，则无需修改；后续编写任何新 Python 脚本也请加上此行。)_

### 二、 配置流程：以 `pathway` 命令为例

#### 第一步：创建无后缀的纯净文件

直接创建一个**没有后缀名**的文件，文件名即为您最终想要使用的命令名。

Bash

```
nano pathway
```

_(在编辑器中粘贴您的脚本代码，然后 `Ctrl+O` 回车保存，`Ctrl+X` 退出)_

#### 第二步：赋予可执行权限（关键）

Linux 系统默认不给文本文件执行权限，必须手动添加，否则系统将其视为普通文本而非程序。

Bash

```
chmod +x pathway
```

#### 第三步：建立并移入个人集中脚本库

为了防止脚本在各个计算目录中混淆或误删，在用户主目录下创建一个专属的脚本库文件夹（如 `my_scripts`），并将刚做好的命令移入其中。

Bash

```
mkdir -p ~/my_scripts   
mv pathway ~/my_scripts/
```

#### 第四步：将脚本库加入系统全局环境变量 PATH

让系统在任何目录都能找到你的自定义命令。只需执行以下命令写入配置（**一劳永逸，以后再加新脚本无需重复此步**）：

Bash

```
echo 'export PATH="$HOME/my_scripts:$PATH"' >> ~/.bashrc
```

#### 第五步：刷新配置使其生效

重新加载配置文件，唤醒刚刚设置的环境变量：

Bash

```
source ~/.bashrc
```

#### 方案优势

1. **彻底原生**：「文件名 = 命令名」，输入 `pathway` 即可直接运行，完全符合 Linux 原生使用习惯。
{ #90cc78}

2. **极强扩展性**：配置过一次 `$PATH` 后，以后写了任何新脚本，只需**去后缀 $\rightarrow$ 赋予 `+x` 权限 $\rightarrow$ 扔进 `~/my_scripts/` 文件夹**，就能立刻成为全局命令，省去了配别名的繁琐。
{ #aa0e91}

> [!note] 核心目的
> 针对实验室本地服务器“无法自动连轴转”、“多用户互相抢占资源”、且“断开终端即停止计算”的痛点，本系统利用 Linux 原生脚本实现了一个零依赖的轻量级排队调度器。
> - **全自动连轴转**：服务器 24 小时满载，自动按顺序调用 48 核执行任务。
> - **超算级体验**：通过 `vsubmit`（提交）、`bjobs`（查看）、`bkill`（终止）三个简单命令管理任务。

### 0.前提条件

> [!attention] 
>  请确保已在 `~` 目录下创建 `my_scripts` 文件夹，并已将其加入系统全局环境变量 `$PATH` 中（详见[[计算相关/脚本的创建与使用（示例）/1.脚本创建及命名\|1.脚本创建及命名]]教程）。

Bash
```
mkdir -p ~/my_scripts   
```

Bash
```
echo 'export PATH="$HOME/my_scripts:$PATH"' >> ~/.bashrc
```

刷新配置使其生效
Bash
```
source ~/.bashrc
```

### 1. 创建全局公共队列目录

需要使用具有 `sudo` 权限的账号，在系统的公共区域创建一个队列文件夹，并赋予所有用户读写权限：

Bash

```
sudo mkdir -p /var/opt/vasp_queue
sudo chmod 777 /var/opt/vasp_queue
```

### 2. 配置提交脚本 `vsubmit`

在 `~/my_scripts/` 下创建文件 `vsubmit`：

Bash

```
nano ~/my_scripts/vsubmit
```

粘贴代码：

Bash

```
#!/bin/bash
# VASP 任务提交命令

QUEUE_DIR="/var/opt/vasp_queue"

if [ ! -d "$QUEUE_DIR" ]; then
    echo "❌ 错误：队列目录 $QUEUE_DIR 不存在，请联系管理员部署。"
    exit 1
fi

TIMESTAMP=$(date +%Y%m%d_%H%M%S_%N)
JOB_FILE="$QUEUE_DIR/${TIMESTAMP}_${USER}.job"

echo "$PWD" > "$JOB_FILE"

echo "✅ 任务已成功加入队列！"
echo "📂 计算路径: $PWD"
echo "🎫 排队凭证: $(basename "$JOB_FILE")"
```

赋予执行权限：`chmod +x ~/my_scripts/vsubmit`

### 3. 配置查看队列脚本 `bjobs`

在 `~/my_scripts/` 下创建文件 `bjobs`：

Bash

```
nano ~/my_scripts/bjobs
```

粘贴代码：

Bash

```
#!/bin/bash
# VASP 队列查看脚本 (带耗时统计版)

QUEUE_DIR="/var/opt/vasp_queue"
RUN_INFO="$QUEUE_DIR/running.info"

echo "📊 VASP 服务器任务监控"
echo "------------------------------------------------"

# 1. 首先显示正在运行的任务
if [ -f "$RUN_INFO" ]; then
    read r_user r_path r_start < "$RUN_INFO"
    now=$(date +%s)
    diff=$((now - r_start))
    
    # 格式化时间
    hours=$((diff / 3600))
    mins=$(( (diff % 3600) / 60 ))
    secs=$((diff % 60))
    
    echo "🏃 [正在运行]"
    echo "👤 提交人: $r_user"
    echo "⌛ 已耗时: ${hours}h ${mins}m ${secs}s"
    echo "📂 路径: $r_path"
else
    echo "💤 [当前空闲] 没有任务正在计算。"
fi

echo "------------------------------------------------"

# 2. 显示排队中的任务
echo "⏳ [等待队列]"
count=0
for job in $(ls -tr "$QUEUE_DIR"/*.job 2>/dev/null); do
    ((count++))
    user=$(echo "$job" | awk -F'_' '{print $NF}' | sed 's/\.job//')
    path=$(cat "$job")
    echo "[$count] 👤 $user | 📂 $path"
done

if [ $count -eq 0 ] && [ ! -f "$RUN_INFO" ]; then 
    echo "✨ 队列干净，随时可以提交新任务！"
elif [ $count -eq 0 ]; then
    echo "目前没有其他人在排队。"
fi
echo "------------------------------------------------"
```

赋予执行权限：`chmod +x ~/my_scripts/bjobs`

### 4. 配置终止任务脚本 `bkill`

在 `~/my_scripts/` 下创建文件 `bkill`：

Bash

```
nano ~/my_scripts/bkill
```

粘贴代码（**严格限定只对当前用户的任务生效，不会误杀他人任务**）：

Bash

```
#!/bin/bash
# VASP 任务终止脚本

QUEUE_DIR="/var/opt/vasp_queue"
echo "🔍 正在检查您的任务状态..."

# 1. 检查是否有正在排队的任务
QUEUED_JOBS=$(ls "$QUEUE_DIR"/*_${USER}.job 2>/dev/null)
if [ -n "$QUEUED_JOBS" ]; then
    rm -f $QUEUED_JOBS
    echo "✅ 已将您的任务从等待队列中移除。"
else
    echo "ℹ️ 您目前没有正在排队的任务。"
fi

# 2. 检查是否有正在运行的任务
if pgrep -u "$USER" vasp > /dev/null; then
    echo "⚠️ 发现您当前有正在运行的 VASP 任务！"
    read -p "是否立即强行终止它？(y/n): " confirm
    if [ "$confirm" = "y" ] || [ "$confirm" = "Y" ]; then
        pkill -u "$USER" vasp
        pkill -u "$USER" mpirun
        echo "🛑 运行中的任务已成功终止！"
        echo "提示：重新提交前建议清理未写完的 WAVECAR 或 CHGCAR。"
    else
        echo "已取消终止操作，任务继续运行。"
    fi
else
    echo "ℹ️ 当前没有属于您的 VASP 计算在运行。"
fi
```

赋予执行权限：`chmod +x ~/my_scripts/bkill`

### 5. 配置核心守护进程脚本 `vdaemon`

在 `~/my_scripts/` 下创建文件 `vdaemon`：

Bash

```
nano ~/my_scripts/vdaemon
```

粘贴代码（**已整合本服务器的 48 核并行命令及解散堆栈限制**）：

Bash

```
#!/bin/bash
# VASP 队列后台守护进程 (带运行时间监控版)

QUEUE_DIR="/var/opt/vasp_queue"
LOG_FILE="/var/opt/vasp_queue/daemon_sys.log"
RUN_INFO="$QUEUE_DIR/running.info"

echo "🚀 VASP 队列守护进程已启动..." >> "$LOG_FILE"

while true; do
    JOB_FILE=$(ls -tr "$QUEUE_DIR"/*.job 2>/dev/null | head -n 1)
    
    if [ -n "$JOB_FILE" ]; then
        TARGET_DIR=$(cat "$JOB_FILE")
        # 提取提交者用户名（从文件名中提取）
        JOB_USER=$(echo "$(basename "$JOB_FILE")" | awk -F'_' '{print $NF}' | sed 's/\.job//')
        rm -f "$JOB_FILE"
        
        if [ -d "$TARGET_DIR" ]; then
            # --- 新增：写入运行实时信息 (格式: 用户 路径 开始秒数) ---
            echo "$JOB_USER $TARGET_DIR $(date +%s)" > "$RUN_INFO"
            chmod 666 "$RUN_INFO" # 确保所有人可见
            
            echo "[$(date +'%Y-%m-%d %H:%M:%S')] 开始执行: $TARGET_DIR" >> "$LOG_FILE"
            cd "$TARGET_DIR" || continue
            
            ulimit -s unlimited
            mpirun -np 48 vasp > vasp_run.log 2>&1
            
            # --- 任务结束，删除运行实时信息 ---
            rm -f "$RUN_INFO"
            echo "[$(date +'%Y-%m-%d %H:%M:%S')] 任务结束: $TARGET_DIR" >> "$LOG_FILE"
        fi
    else
        sleep 30
    fi
done
```

赋予执行权限：`chmod +x ~/my_scripts/vdaemon`

### 6. 启动排队系统（一劳永逸）

在任意终端执行一次以下命令将守护进程挂载到后台。即使断开远程连接或关闭本地终端，服务器也会全自动接管后续的所有任务：

Bash

```
nohup vdaemon > /dev/null 2>&1 &
```

### 一、 自动化提取脚本：`tiqvnengliang`

> [!note] 
> 这个脚本的核心使命是“自动化的高通量数据提取”。在第一性原理计算中，面对成百上千个计算文件夹，手动打开文本提取能量极易出错且耗时。这个脚本就像一个自动巡逻的机器人，帮你完成这一切。

#### 第一步：创建脚本

在 `~/my_scripts/` 目录下创建该文件：

Bash

```
nano ~/my_scripts/tiqvnengliang
```


> [!attention] 
> 若没有该文件夹则创建一个 

Bash
```
mkdir -p ~/my_scripts
```

#### 第二步：粘贴以下代码

Python

```
import os
import csv
import re

def parse_vasp_directory(dir_path):
    """
    解析单个包含 VASP 计算文件的目录，提取状态、能量和磁矩。
    """
    outcar_path = os.path.join(dir_path, 'OUTCAR')
    oszicar_path = os.path.join(dir_path, 'OSZICAR')

    status = "文件夹为空"
    energy = "-"
    mag = "-"

    # 如果连 OUTCAR 都没有，直接返回空状态
    if not os.path.exists(outcar_path):
        return status, energy, mag

    # 只要有 OUTCAR 但还没检测到收敛标志，就默认设为报错终止或未完成
    status = "报错终止/未完成"
    
    try:
        # 1. 解析 OUTCAR 提取能量和状态
        with open(outcar_path, 'r', encoding='utf-8', errors='ignore') as f:
            lines = f.readlines()
            
        found_energy = False
        found_status = False
        
        # 【技巧】：倒序读取（从文件末尾往上读）
        # 因为 VASP 的收敛标志和最终能量都在文件末尾，倒序读取能极大提升速度
        for line in reversed(lines):
            # 检查是否正常收敛
            if "reached required accuracy" in line and not found_status:
                status = "已完成"
                found_status = True
                
            # 提取最后一次的 energy(sigma->0)
            if "energy(sigma->0)" in line and not found_energy:
                # 使用正则表达式提取等号后面的数字（支持正负号和小数点）
                match = re.search(r'energy\(sigma->0\)\s*=\s*([-+0-9.]+)', line)
                if match:
                    energy = match.group(1)
                    found_energy = True
            
            # 如果两个都找到了，就不需要继续往上读了，节约时间
            if found_energy and found_status:
                break

        # 2. 解析 OSZICAR 提取最后一步的磁矩
        if os.path.exists(oszicar_path):
            with open(oszicar_path, 'r', encoding='utf-8', errors='ignore') as f:
                osz_lines = f.readlines()
                if osz_lines:
                    last_line = osz_lines[-1] # 直接抓取最后一行
                    # 匹配 mag= 后面的数字
                    match = re.search(r'mag=\s*([-+0-9.]+)', last_line)
                    if match:
                        mag = match.group(1)
                        
    except Exception as e:
        status = f"文件读取错误"

    return status, energy, mag


def main():
    # 获取当前脚本所在的运行目录
    root_dir = os.getcwd()
    output_csv = "VASP_Results_Summary.csv"
    
    print(f"开始扫描目录: {root_dir}")
    print("正在努力提取数据，请稍候...\n")

    results = []

    # os.walk() 会像树根一样向下遍历所有子文件夹
    for current_dir, sub_dirs, files in os.walk(root_dir):
        # 核心逻辑：只有当该文件夹内存在 OUTCAR 时，才认为这是一个计算节点
        if 'OUTCAR' in files:
            # 获取相对路径，方便我们拆分“大类结构”、“吸附模型”
            rel_path = os.path.relpath(current_dir, root_dir)
            path_parts = rel_path.split(os.sep)
            
            # 灵活处理目录层级结构
            folder_level_1 = path_parts[0] if len(path_parts) > 0 else "-"
            folder_level_2 = path_parts[1] if len(path_parts) > 1 else "-"
            folder_level_3 = path_parts[2] if len(path_parts) > 2 else "-"
            # 如果深度超过3层，将后面的路径拼接到Level_3中
            if len(path_parts) > 3:
                folder_level_3 = os.path.join(*path_parts[2:])

            # 调用提取函数
            status, energy, mag = parse_vasp_directory(current_dir)
            
            # 保存到结果列表
            results.append([folder_level_1, folder_level_2, folder_level_3, status, energy, mag])
            print(f"已处理: {rel_path} -> 状态: {status}")

    # 将结果写入 CSV 表格
    if results:
        with open(output_csv, 'w', newline='', encoding='utf-8-sig') as csvfile:
            writer = csv.writer(csvfile)
            # 写入表头
            writer.writerow(['大类结构 (Level 1)', '吸附模型 (Level 2)', '具体构型 (Level 3)', '状态', '能量 (F=, eV)', '磁矩 (mag=)'])
            # 写入数据
            writer.writerows(results)
        print(f"\n✅ 数据提取完毕！已成功生成表格文件：{output_csv}")
    else:
        print("\n⚠️ 扫描结束，当前目录及其子目录下未发现任何包含 OUTCAR 的文件夹。")

if __name__ == "__main__":
    main()
```


#### 第三步：给文件添加可执行权限（关键步骤，缺了必报错）

Linux 系统默认不给文本文件执行权限，必须手动添加：

bash

```
chmod +x ~/my_scripts/tiqvnengliang
```


#### 第四步：**把脚本库加入系统全局环境变量 PATH（一劳永逸）**

让系统在任何目录都能找到你的`tiqvnengliang`命令，执行以下命令写入配置：

bash(若该目录之前已经加入[PATH](2.脚本创建及命名.md#^39e1b2)过则跳过该步骤)

```
echo 'export PATH="$HOME/my_scripts:$PATH"' >> ~/.bashrc
```

#### 第五步：刷新配置生效

bash

```
source ~/.bashrc
```

---


> [!caution] 
>  
脚本文件格式应为`UTF-8`，若在 Windows 下编辑过，可能变为 GBK 编码导致乱码报错，须输入以下命令转换格式：

Bash

```
iconv -f GBK -t UTF-8 ~/my_scripts/tiqvnengliang -o ~/my_scripts/tiqvnengliang
```

_(这行代码的意思是：把该文件从 GBK 编码读取，转换为 UTF-8，然后重新输出并覆盖原文件)_
